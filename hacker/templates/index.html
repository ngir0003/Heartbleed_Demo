How it works
1
Victim logs in
Open SecureBank (port 4001) and sign in as alice or bob. Their session cookie is now stored in server heap memory.
2
Send malformed heartbeat
We send 1 byte of real payload but claim 0x4000 (16,384 bytes). OpenSSL does not check the length.
3
Server leaks memory
OpenSSL copies our 1 byte then reads 16,383 extra bytes straight off the heap — whatever is adjacent.
4
Find session token
We scan the raw dump for session=. The victim's cookie may appear in the leaked data.
5
Hijack session
Paste the cookie into your browser. Reload SecureBank — you're logged in as the victim, no password needed.

              Double-click the empty row at the bottom of the cookie table. Set Name = session and Value = the token you copied above. The page will still show /login — that is expected, do not reload yet.
            

          


          

            
5
